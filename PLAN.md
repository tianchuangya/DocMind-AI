# DocMindAI 桌面文本编辑器开发计划

## 概要

以 **C++ / Qt 6** 开发本地优先的跨平台桌面应用。首期以 Markdown 为文档主格式，提供源码编辑与实时预览、DOCX/PDF/HTML 导入导出、兼容 OpenAI API 的写作助手及本地知识库问答。同步仅定义扩展接口，不接入真实云端服务。

## 架构与关键模块

- 使用 Qt Widgets 构建主窗口、标签页、文件树、编辑器、预览和 AI 侧栏；使用 Qt WebEngine 承载 HTML 预览。
- 按三层拆分：`UI/交互层` → `应用服务层` → `本地存储、文件系统、转换与模型接口`，避免界面直接调用外部命令或网络。
- 采用 C++20、Qt 6 Core/Gui/Widgets/Network/Sql/WebEngine，SQLite 保存元数据、设置、全文索引与向量数据。
- 敏感 API Key 不写入 SQLite：通过系统钥匙串保存；数据库仅保存密钥引用、模型配置和非敏感设置。

## 模块 A：编辑与预览核心

- 实现多标签文档工作区：新建、打开、保存、另存为、关闭确认、最近文件、自动保存和崩溃恢复。
- 以 Markdown 源码编辑为主：行号、语法高亮、撤销/重做、查找替换、快捷键、脏状态提示和 UTF-8 文件读写。
- 使用 Markdown 渲染服务将编辑内容防抖转换为 HTML，注入统一 CSS 主题后交给 WebEngine 实时预览。
- 支持编辑区与预览区的滚动同步、编辑/预览分栏切换、浅色/深色主题，以及图片、表格、代码块、任务列表、脚注等常用 Markdown 语法。
- 定义内部 `DocumentSession` 接口：文档路径、内容、编码、修改状态、光标位置、渲染版本和保存状态均由它统一管理，预览和导出只消费其稳定快照。

## 模块 B：文档转换中枢

- 将转换实现为异步、可取消的 `ConversionService` 和任务队列；任务输入为源文件/当前文档快照、目标格式、输出路径与资源路径，输出为状态、日志、产物路径和可读错误。
- 随安装包捆绑 Pandoc：
  - Markdown ↔ DOCX；
  - Markdown ↔ HTML；
  - DOCX、HTML 导入后统一转换为 Markdown。
- 随安装包捆绑 Tectonic，供 Pandoc 将 Markdown 导出为 PDF；导出时保留标题、段落、表格、代码块、图片及基础主题样式。
- 随安装包捆绑 Poppler 文本提取组件：PDF 导入为 Markdown 时以“提取后的文本/基础结构”为目标，不承诺还原复杂版式、表格和扫描件；扫描 PDF 明确提示需后续 OCR 能力。
- 统一处理资源文件：导入图片复制/引用策略、导出时相对路径解析、临时目录清理、目标文件覆盖确认及失败日志展示。
- 通过转换能力探测页显示捆绑工具版本和健康状态，工具缺失或进程失败时提供明确诊断信息。

## 模块 C：AI 服务与数据管理

- 提供兼容 OpenAI API 的可配置客户端：Base URL、API Key、聊天模型、嵌入模型、超时和代理设置；请求采用后台任务、支持取消、流式输出及统一错误映射。
- 写作助手基于当前选区或全文提供续写、改写、润色、摘要、翻译和自定义指令；所有结果先在 AI 面板展示，由用户显式选择插入、替换或复制。
- 知识库支持导入 Markdown、DOCX、PDF：
  - 经转换/提取管道获取纯文本；
  - 按标题与长度分块，保留来源文件、页码/段落定位和内容哈希；
  - 生成嵌入后存入本地 SQLite，结合 FTS5 关键词召回和向量余弦相似度排序；
  - 将命中文本及来源作为上下文发送给模型，回答界面展示可点击的来源引用。
- 建立 `AIProvider`、`EmbeddingProvider`、`KnowledgeRepository`、`SyncProvider` 抽象接口。首期实现 OpenAI-compatible Provider 和本地 Repository；`SyncProvider` 仅提供变更记录、拉取、推送与冲突结果的数据契约，不实现云同步。
- 本地数据包含应用设置、文档恢复记录、知识库条目、分块、向量、导入状态和操作日志；提供清除单个知识库、重建索引和清除全部 AI 本地数据的界面入口。

## 测试与验收

- 为 Markdown 渲染、文档会话、文件编码、自动保存恢复、转换参数生成、任务取消、SQLite 仓储、分块和检索排序编写单元测试。
- 使用 Markdown、含图片/表格的 DOCX、文本型 PDF、复杂 PDF、HTML 五类夹具进行端到端导入导出测试。
- 验证 DOCX/HTML 往返后核心标题、段落、列表、代码块、表格和图片引用可用；验证 PDF 导出可打开且样式正常。
- 使用模拟 OpenAI 服务验证流式响应、超时、认证失败、取消请求和用户确认后才写入编辑器。
- 验证知识库可跨重启保留、删除来源后不再被检索、引用定位正确，并在无网络/无 API Key 时保持编辑和转换功能可用。
- 首期验收标准：用户可离线完成 Markdown 编辑、实时预览、DOCX/PDF/HTML 导入导出；配置兼容 API 后可使用写作助手并基于导入资料获得带来源的问答结果。

## 默认假设

- 首期不包含账号体系、真实云同步、协作编辑、OCR、PDF 版式级反向还原或 Word 式所见即所得编辑。
- PDF 导入面向可提取文本；扫描件和复杂排版属于后续 OCR/高级解析迭代。
- 应用将以本地文件和 SQLite 为权威数据源；同步接口预留但不影响离线使用。
- 第三方转换与 PDF 排版工具随应用发布，并在打包阶段分别验证 macOS、Windows、Linux 的可执行性与许可证要求。
