### 对话 2: 程序无法启动问题 (20:30 - 20:50)

**问题描述**: 
用户反馈"双击运行不了"

**诊断过程**:
1. 发现进程在运行但没有窗口显示
2. 添加调试输出追踪初始化流程
3. 发现问题：Logger 中的 QMutex 死锁

**根本原因**:
```cpp
void Logger::init() {
    QMutexLocker locker(&m_mutex);  // 获取锁
    // ...
    info("Logger", "初始化完成");   // 调用 log() → 再次尝试获取锁 → 死锁！
}
```

**解决方案**:
将 `QMutex` 改为 `QRecursiveMutex`（递归互斥锁），允许同一线程多次获取锁。

```cpp
// Logger.h
QRecursiveMutex m_mutex;  // 从 QMutex 改为 QRecursiveMutex
```

**结果**: ✅ 程序正常启动

### 对话 3: Markdown 预览不渲染 (20:50 - 21:10)

**问题描述**: 
用户反馈"显示视图还是有问题"，预览区不显示渲染后的 Markdown

**诊断过程**:
1. 添加详细的调试输出
2. 追踪信号链：`textChanged` → `updatePreview` → `setMarkdownContent` → `doRender`
3. 发现 `onCurrentTabChanged` 没有被调用

**根本原因**:
```cpp
// TabManager::openFile()
int index = m_tabWidget->addTab(editor, title);  // 触发 currentChanged 信号
m_tabs.append(info);  // 但此时 m_tabs 还是空的！

// onCurrentChanged 被调用时
void onCurrentChanged(int index) {
    if (index < m_tabs.size()) {  // 条件失败，信号链断裂
        emit currentTabChanged(...);
    }
}
```

**解决方案**:
使用 `QSignalBlocker` 阻止 `addTab` 期间触发信号，先添加到 `m_tabs`，再添加标签。

```cpp
TabInfo info;
m_tabs.append(info);  // 先添加

{
    QSignalBlocker blocker(m_tabWidget);  // 阻止信号
    m_tabWidget->addTab(editor, title);
}

emit currentTabChanged(session);  // 手动触发
```

**结果**: ✅ 预览正常渲染

### 对话 4: 滚动同步功能 (21:10 - 21:20)

**用户需求**: 
"你不如做一个上拉列表可以勾选，顺便添加缩放"

**实现功能**:
1. **下拉菜单控制同步模式**:
   - 双向同步（默认）
   - 编辑器→预览
   - 预览→编辑器
   - 关闭同步

2. **预览区缩放**:
   - Ctrl + 滚轮 = 缩放
   - 向上滚 = 放大（最大 72pt）
   - 向下滚 = 缩小（最小 8pt）

**核心代码**:
```cpp
// ScrollSyncManager.h
void setEditorToPreviewEnabled(bool enabled);
void setPreviewToEditorEnabled(bool enabled);

// PreviewWidget.cpp - 缩放功能
void PreviewWidget::wheelEvent(QWheelEvent* event) {
    if (event->modifiers() & Qt::ControlModifier) {
        int delta = event->angleDelta().y();
        QFont f = font();
        int size = f.pointSize();
        
        if (delta > 0 && size < 72) {
            f.setPointSize(size + 1);
            setFont(f);
            if (!m_currentMarkdown.isEmpty()) {
                doRender();  // 重新渲染
            }
        } else if (delta < 0 && size > 8) {
            f.setPointSize(size - 1);
            setFont(f);
            if (!m_currentMarkdown.isEmpty()) {
                doRender();
            }
        }
    }
}
```

**结果**: ✅ 功能完成

### 对话 5: 代码块样式问题 (21:20 - 21:40)

**问题描述**: 
用户反馈"代码块整体深色底色；每一行 Python 代码单独带深色行底色；两行代码的行间隙处多出一块不一样的中间背景色块"

**问题分析**:
三层背景色不一致导致割裂感：
```
┌─ .code-block (背景 A) ─────┐
│  ┌─ pre (背景 B) ─────────┐ │
│  │  ┌─ code (背景 C) ─┐   │ │  ← 行间间隙显示 body 背景
│  │  │ 第1行代码       │   │ │
│  │  │ ── 突兀横条 ──  │   │ │  ← 割裂感
│  │  │ 第2行代码       │   │ │
│  │  └─────────────────┘   │ │
│  └────────────────────────┘ │
└─────────────────────────────┘
```

**解决过程**:

**尝试 1**: 三层使用相同背景色
- ❌ 问题依旧，行间间隙仍显示 body 背景

**尝试 2**: code 使用 `display: inline` + `transparent`
- ❌ QTextBrowser 对 `transparent` 支持不好

**尝试 3**（用户建议）: 取消行间距
- ✅ 设置 `line-height: 1.0`，消除行间间隙

**最终方案**:
```css
.code-block pre {
    line-height: 1.0;  /* 从 1.5 改为 1.0 */
    background-color: #1e1e2e;
}
.code-block code {
    line-height: 1.0;  /* 从 1.5 改为 1.0 */
    background: none;
    display: inline;
}
```

**结果**: ✅ 代码块背景完全连续，无割裂感

### 对话 6: 浅色主题文字不可见 (21:40 - 21:50)

**问题描述**: 
用户反馈"白色主题那个菜单栏等等都是过白然后就看不清文字了"

**根本原因**:
浅色主题的 QSS 样式中没有为 UI 元素设置明确的文字颜色，导致使用了系统默认颜色（可能是浅色），与浅色背景对比度不够。

**解决方案**:
为所有 UI 元素添加明确的深色文字颜色：

```cpp
// ThemeManager.cpp - loadLightQSS()
QMenuBar {
    background-color: #f0f0f0;
    color: #333333;  /* 添加文字颜色 */
}
QMenu::item {
    color: #333333;  /* 添加文字颜色 */
}
QToolBar {
    color: #333333;  /* 添加文字颜色 */
}
QStatusBar {
    color: #555555;  /* 添加文字颜色 */
}
QLabel, QCheckBox, QComboBox {
    color: #333333;  /* 添加文字颜色 */
}
```

**结果**: ✅ 浅色主题下所有文字清晰可见

### 对话 7: 代码块自适应宽度 (21:50 - 22:00)

**用户需求**: 
"代码块背景右边距离取文本最长行的长度，再右边就是主题色"

**尝试方案**:
```css
.code-block {
    display: inline-block;  /* 自适应宽度 */
    max-width: 100%;
}
```

**问题**:
QTextBrowser 对 `display: inline-block` 支持不好，代码块仍然是全宽。

**结论**:
QTextBrowser 基于 QTextDocument，对 CSS 的支持有限，无法实现代码块宽度自适应到最长行。

**妥协方案**:
保持代码块全宽，但调整配色：
- 白色主题：浅灰色背景 (#f5f5f5)
- 深色主题：白色低透明背景 (rgba(255,255,255,0.05))

**额外改进**:
- 移除语言标识和代码之间的间距
- 移除分隔线

```css
.code-block .code-lang {
    margin-bottom: 0;
    border-bottom: none;
}
.code-block pre {
    margin-top: 0;
}
```

**结果**: ⚠️ 部分实现（宽度自适应失败，但配色和间距改进成功）

## 技术难点总结

### 1. Qt6 兼容性问题
| 问题 | Qt5 | Qt6 | 解决方案 |
|-----|-----|-----|---------|
| QTextCodec | ✅ 可用 | ❌ 已移除 | 使用 QStringDecoder |
| setFontFamily() | ✅ 可用 | ❌ 已废弃 | 使用 setFontFamilies() |
| fromUtf16() | ushort* | char16_t* | 更改类型转换 |
| addAction() | 4参数 | 签名变化 | 使用 lambda 或分步调用 |

### 2. QTextBrowser 限制
- ❌ 不支持完整的 HTML 文档结构（只能渲染 body 内容）
- ❌ 对 `display: inline-block` 支持不好
- ❌ 对 `transparent` 背景支持有限
- ✅ 支持 `setDefaultStyleSheet()` 设置 CSS
- ✅ 支持基本的 HTML 标签和 CSS 属性

### 3. 信号与槽问题
- **QTabWidget::addTab()** 会立即触发 `currentChanged` 信号
- **解决方案**: 使用 `QSignalBlocker` 阻止信号，手动触发

### 4. 多线程与死锁
- **QMutex** 不支持递归锁定
- **解决方案**: 使用 `QRecursiveMutex`

## 经验教训

### 1. QTextBrowser vs QWebEngineView
- **QTextBrowser**: 轻量，但 CSS/HTML 支持有限
- **QWebEngineView**: 完整支持，但需要额外安装
- **教训**: 预览功能建议直接使用 QWebEngineView

### 2. 信号与槽的时序问题
- `addTab()` 会立即触发信号
- **教训**: 先准备数据，再触发 UI 更新，必要时使用 `QSignalBlocker`

### 3. 调试技巧
- 使用 `std::cerr` 输出调试信息（WIN32 应用需要手动输出到 stderr）
- 安装自定义消息处理器：`qInstallMessageHandler()`
- 逐步追踪信号链，找出断裂点

### 4. CSS 在 QTextBrowser 中的限制
- 不支持 `display: inline-block`
- `transparent` 背景可能不生效
- **解决方案**: 使用明确的背景色，避免依赖继承
